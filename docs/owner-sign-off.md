# Раздел 2 · UAT — критерии приёмки / 2-bo'lim · Qabul mezonlari

## 2.1 Длительность UAT

UAT идёт **2–4 недели** на одной пилотной ферме (площадка определяется
заказчиком до старта). За это время проверяется реальная эксплуатация
операторами и руководителями.

## 2.2 Критерии успеха (все 4 обязательны)

1. ✅ **≥ 80% daily-log заполнены до 12:00** — на основе данных
   `apps.feedlot.daily_log_check_task` за последние 7 дней.
2. ✅ **< 5% записей требуют коррекции** в первые 24 часа после ввода
   (через `audit_log` смотрим UPDATE-операции).
3. ✅ **Среднее время заполнения формы ≤ 2 минут** для оператора в полевых
   условиях (хронометраж на 5 операторах × 5 заполнений).
4. ✅ **Zero P0/P1 багов** в последние 7 дней UAT (P0 = блокирует работу
   всей фермы, P1 = блокирует один модуль).

При выполнении всех 4 — UAT считается пройденным, система переходит
в production.

## 2.3 Если критерии не выполнены

Подрядчик дорабатывает Mobile/UX до достижения критериев. Дополнительной
платы за фиксы UAT-замечаний нет.

# Раздел 3 · Обязательства заказчика / 3-bo'lim · Buyurtmachining majburiyatlari

## 3.1 Что должен предоставить заказчик до старта Этапа 1

- [ ] Определена пилотная ферма (название, адрес, поголовье)
- [ ] Назначен ответственный за daily-log (оператор + супервайзер)
- [ ] Решение по устройствам операторов: свои телефоны или закупка
- [ ] Telegram-рабочие группы для алертов созданы и заказчик доступен
- [ ] Языки интерфейса: только UZ или UZ+RU
- [ ] Доступ к интернету в местах работы операторов (Wi-Fi/4G)
- [ ] Заведены справочники: породы, кормосмеси, поставщики (или
  передаются файлом для миграции)

## 3.2 Доступ для разработчика

- [ ] Тестовый сервер / staging-окружение
- [ ] Production-сервер (при готовности к rollout)
- [ ] Доменное имя для production
- [ ] Доступ к Telegram-боту (@BotFather token)

# Раздел 4 · Обязательства подрядчика / 4-bo'lim · Pudratchi majburiyatlari

- ✅ Поставка системы в работающем состоянии (см. Раздел 1)
- ✅ Развёртывание на staging и production по согласованному графику
- ✅ Обучение администратора и 1–2 операторов (≤ 1 день, RU)
- ✅ Документация:
  - Руководство администратора (RU)
  - Краткая шпаргалка для операторов (UZ + RU, 1–2 страницы)
- ✅ Поддержка в течение UAT (любые баги — без доплаты)
- ✅ Telegram-канал для срочной коммуникации в часы работы фермы
- ✅ Резервная копия БД ежедневно (хранение ≥ 14 дней)

# Раздел 5 · Что после UAT / 5-bo'lim · UAT'dan keyin

После успешного прохождения UAT:

- 📌 Система переходит в production
- 📌 Заказчик получает административный доступ для self-service
  (новые пользователи, роли, профили усушки)
- 📌 Подрядчик переходит на режим **сопровождения** по отдельному договору
  (объём, SLA и стоимость согласуются дополнительно)
